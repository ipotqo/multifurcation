The {\ttfamily multifurcation} skeleton is a convention for a {\ttfamily cmake} structure with branched/nested subdivisions; it is based on the \href{https://github.com/vector-of-bool/pitchfork}{\texttt{ Pitchfork}} but with several modifications as discussed below. Consider this to be a starter kit for a {\ttfamily C++20} codebase that features numerous smaller and self-\/contained libraries as well as linter ({\ttfamily clang-\/format} \& {\ttfamily clang-\/tidy}) and debugger ({\ttfamily gdb}) capabilities, documentation ({\ttfamily Doxygen}), containerisation ({\ttfamily Docker}), unit tests ({\ttfamily Google\+Test}) and CI/\+CD functionalities ({\ttfamily Git\+Hub Actions}).
